# AAD

## Purpose
Document canonical Additional Authenticated Data (AAD) construction for message encryption/decryption.

## Current Implementation
- AAD is generated by `AadCodec.buildAAD(EncryptedMessage)`.
- Fields are encoded deterministically (length-prefixed UTF-8 strings + big-endian numeric fields).
- Decrypt path reconstructs AAD from DTO metadata; it does not trust serialized `aadB64` input.
- Field order is fixed: `version`, `algorithm`, `senderId`, `recipientId`, `timestampEpochMs`, `ttlSeconds`, `contentType`, `contentLength`.

## Key Types/Interfaces
- `shared.crypto.AadCodec`
- `shared.dto.EncryptedMessage`

## Flow
1. Collect metadata fields (`version`, `algorithm`, sender/recipient ids, timestamp, ttl, content type, content length).
2. Encode strings with 4-byte length prefixes and UTF-8 bytes.
3. Append numeric fields as fixed-width longs.
4. Use AAD during AES-GCM encrypt/decrypt for integrity binding.

## Error/Security Notes
- Any metadata mutation changes AAD and should fail GCM authentication on decrypt.
- Deterministic encoding avoids sender/receiver drift.
